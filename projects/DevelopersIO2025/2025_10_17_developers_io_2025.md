# Hapio AIとは

![w:900px](./images/hapio-overview.png)

AIがあなたのプレゼンテーションを多角的に分析・評価し、具体的な改善提案を行うプレゼンテーション練習プラットフォーム

---

# 解決したい課題

## プレゼンテーションスキルの重要性

近年、プレゼンテーションはますます重要性が上がっています。

- 人前で説得力のある質の高いプレゼンを行いたい
- 自分の言葉遣いや発言の適切性を知りたい
- プレゼンテーションの構成や論理展開を改善したい
- 発表前にプレゼンテーションの練習をしたい

## AIが発展する中、より人間的スキルが求められる

プレゼンテーションは必須のスキルになります。

---

# 主な機能

## シチュエーション設定

あなたの想定するプレゼンテーションのシチュエーションを選択

- **Topic** - 発表テーマ
- **Scene** - 発表場面
- **Audience Type** - 聴衆の種類
- **Formality Level** - フォーマル度
- **Language** - 使用言語
- **Presentation Purpose** - 発表目的

また、**制限時間**も設定可能で、時間管理も実施できます。

---

# 多角的評価システム

## 5つの観点から総合評価

- **Content Accuracy** - 内容の正確性・適切性
- **Language & Wording** - 言葉遣い・語彙選択の適切性
- **Situation Appropriateness** - シチュエーションとの整合性
- **Facial Expression** - 表情・非言語コミュニケーション
- **Structure & Logic** - 構成・論理展開の明確性

## 分析内容

- 音声から文字起こし
- 顔の表情を分析
- シチュエーション、スライド資料、プレゼン内容を総合的に評価
- スライド単位での評価 + 全体を通した総合評価

# ソースコードのアーキテクチャ

![w:600px](./images/architecture-diagram.png)

## フロントエンド: Feature-Based

- `/src/app/_client/features/` 配下に機能単位でディレクトリを配置
- `home/`, `project/`, `projects/`, `settings/`

## バックエンド: クリーンアーキテクチャ

- **domain層** - エンティティ、ビジネスルール
- **use-case層** - アプリケーションロジック
- **infrastructure層** - 外部システム連携
- **presentation層** - API
- **ESLintで依存関係を管理**

---

<!-- _class: small-text -->

# ディレクトリ構成

```
/src/app
├─ _client/                        # クライアントサイドのコード
│  ├─ common/                      # 共通コンポーネント
│  └─ features/                    # 機能単位のディレクトリ
│     ├─ home/                     # ホームページ
│     ├─ project/                  # プロジェクト詳細（プレゼン練習画面）
│     ├─ projects/                 # プロジェクト一覧
│     └─ settings/                 # 設定ページ
├─ _server/                        # サーバーサイドのコード
│  ├─ domain/                      # ドメイン層（エンティティ、ビジネスルール）
│  │  ├─ core/                     # コアドメインモデル
│  │  │  ├─ */model/               # ドメインモデル
│  │  │  ├─ */domain-function/     # ドメインサービス
│  │  │  └─ */repository/          # リポジトリ
│  │  ├─ master-data/              # マスターデータ
│  │  └─ support/                  # APIクライアントなどのサポート機能
│  ├─ use-case/                    # ユースケース層
│  ├─ infrastructure/              # インフラストラクチャ層
│  ├─ presentation/                # プレゼンテーション層
│  ├─ di-container/                # 依存性注入コンテナ
│  └─ util/                        # ユーティリティ
└─ (routes)/                       # Next.js App Routerのルーティング
   ├─ (site)/                      # 認証済みユーザーのみ閲覧可能
   │  ├─ projects/                 # プロジェクト一覧
   │  ├─ settings/                 # 設定ページ
   │  └─ layout.tsx
   └─ (static)/                    # 未認証ユーザーでも閲覧可能
      ├─ page.tsx                  # トップページ
      ├─ auth/                     # 認証関連
      ├─ privacy/                  # プライバシーポリシー
      └─ terms/                    # 利用規約
```

# 重要なポイント

## 個人開発だからといってアーキテクチャを疎かにしない

- **クリーンアーキテクチャを採用**
- **ESLintで依存関係を管理**
- **serverとclientを分離し、実行環境を明確化**

## なぜアーキテクチャが重要か

- AIエージェントが判断しやすくなる
- コードの品質を保ちながら開発できる
- 保守性・拡張性が向上

## → AI時代だからこそ、設計が重要

---

# AI Agentを使う上での注意点

## AIだけでは完結しない

- **要件定義は自分で行う必要がある**
- デザインは別途必要
- 最終的な品質チェックは人間が行う

## アーキテクチャの重要性

- 整備されたアーキテクチャがあってこそ、AIが真価を発揮
- 統一されたコーディングルールが必要
- ディレクトリ構造を明確にする
